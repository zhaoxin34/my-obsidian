# AntAgent 架构详解

## 1. 消息流转图

```mermaid
sequenceDiagram
    participant User as 用户
    participant Channel as 接入渠道
    participant MB as Message Bus
    participant Agent as Agent Loop
    participant Tools as Tools Registry
    participant Provider as LLM Provider
    participant Memory as Memory System

    User->>Channel: 发送消息
    Channel->>MB: publish("message", msg)

    MB->>Agent: 投递消息
    Agent->>Agent: 1. 构建上下文

    Agent->>Memory: 获取相关记忆
    Memory-->>Agent: 返回记忆内容

    Agent->>Provider: 发送完整上下文
    Provider-->>Agent: LLM 响应

    alt 有工具调用
        Agent->>Tools: 执行工具
        Tools-->>Agent: 工具结果
        Agent->>Provider: 发送工具结果
        Provider-->>Agent: 下一轮响应
    end

    Agent-->>Channel: 返回最终响应
    Channel-->>User: 显示回复

    Agent->>Memory: 保存对话到记忆
```

## 2. 工具执行流程

```mermaid
flowchart TD
    A[用户请求] --> B{是否需要工具?}

    B -->|Yes| C[解析工具调用]
    B -->|No| Z[直接返回响应]

    C --> D{工具是否存在?}
    D -->|No| E[返回错误: 工具未找到]
    D -->|Yes| F{参数是否正确?}

    F -->|No| G[返回错误: 参数错误]
    F -->|Yes| H{安全检查}

    H -->|No| I[返回错误: 权限不足]
    H -->|Yes| J[执行工具]

    J --> K{执行成功?}
    K -->|No| L[返回错误信息]
    K -->|Yes| M[格式化结果]

    M --> N[添加到上下文]
    N --> O[再次调用 LLM]
    O --> P{需要更多工具?}

    P -->|Yes| C
    P -->|No| Q[返回最终响应]

    E --> Z
    G --> Z
    I --> Z
    L --> Z
    Q --> Z
```

### 7.2 Provider 接口

```python
# antagent/providers/base.py
from abc import ABC, abstractmethod
from typing import List, Dict, Any
from dataclasses import dataclass

@dataclass
class Message:
    role: str
    content: str
    tool_calls: List[Any] = None

@dataclass
class Response:
    content: str
    tool_calls: List[Any] = None

class LLMProvider(ABC):
    @abstractmethod
    async def chat(self, messages: List[Message]) -> Response:
        """发送聊天请求"""
        pass

    @abstractmethod
    async def get_available_models(self) -> List[str]:
        """获取可用模型"""
        pass
```
