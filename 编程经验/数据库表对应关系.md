
| 关系类型     | 表达方式        | 含义说明               | 示例场景         | 是否常见   | 建模建议                    |
| -------- | ----------- | ------------------ | ------------ | ------ | ----------------------- |
| 一对一（必选）  | 1 : 1       | 每个A必须有一个B，B也必须有一个A | 用户 ↔ 身份证     | ❌ 较少   | 可合表，或加唯一外键              |
| 一对一（可选）  | 1 : 0..1    | A可以没有B，但最多一个B      | 用户 → 用户扩展信息  | ✅ 很常见  | 子表 user_id + UNIQUE     |
| 一对多（必选）  | 1 : 1..N    | A至少有一个B            | 订单 → 订单项     | ❌ 较少   | 子表外键 NOT NULL           |
| 一对多（可选）  | 1 : 0..N    | A可以没有B，也可以有多个B     | 用户 → 订单      | ✅ 非常常见 | 标准外键关系                  |
| 多对一      | N : 1       | 多个A属于一个B（1:N反向）    | 订单 → 用户      | ✅ 非常常见 | 外键在“多”的一侧               |
| 多对多      | N : M       | 多个A对应多个B           | 用户 ↔ 角色      | ✅ 非常常见 | 中间表（关联表）                |
| 多对多（带属性） | N : M (+属性) | 关系本身有字段            | 用户 ↔ 课程（含成绩） | ✅ 常见   | 中间表加字段                  |
| 自关联（一对多） | 1 : N（自身）   | 表自己关联自己            | 部门树 / 评论回复   | ✅ 常见   | parent_id               |
| 自关联（多对多） | N : M（自身）   | 自己和自己多对多           | 用户好友关系       | ✅ 常见   | 中间表 user_id + friend_id |
 ## 关键术语
| 概念                | 含义                 |
| ----------------- | ------------------ |
| Cardinality（基数）   | 1:1 / 1:N / N:M    |
| Optionality（可选性）  | 0..1 / 0..N / 1..N |
| Association（关联）   | 表之间的关系             |
| Foreign Key（外键）   | 建立关系的字段            |
| Unique Constraint | 用于保证 1:1           |
