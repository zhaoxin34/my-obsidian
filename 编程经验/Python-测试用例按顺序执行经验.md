## Python 测试用例按顺序执行

### 背景

在自动化 E2E 测试中，某些测试用例有先后依赖关系，例如：
- 注册测试需要最先执行，确保测试账号可用
- 登录测试依赖注册
- 业务功能测试依赖登录

pytest 默认按文件名字母顺序执行测试，但这种方式不够灵活，无法自定义顺序。

### 目标

实现测试用例按指定顺序执行。

### 方案

将测试文件按执行顺序添加数字前缀：

```
test_001_register.py      # 第1个
test_002_cdp_log.py       # 第2个
test_003_cdp_home.py      # 第3个
test_004_cdp_edge.py      # 第4个
test_005_cdp_customer.py  # 第5个
```

pytest 默认按文件名字母顺序收集执行，这样就实现了固定顺序。

**优点**：
- 简单直接，无需额外配置
- 文件名清晰，易于理解执行顺序

**注意**：
- 数字前缀用 001、002、003... 格式，确保正确排序
- 建议间隔留余量，方便中间插入新测试

### 验证顺序

查看测试收集顺序：
```bash
pytest --collect-only -q
```
