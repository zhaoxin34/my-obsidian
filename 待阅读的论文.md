https://arxiv.org/abs/2605.27366 1. 论文核心: MUSE-Autoskill 把 Agent 的技能从一次性提示或脚本，升级为可创建、可记忆、可管理、可测试、可修补的长期资产。 2. 问题背景: 现有自动技能系统常把技能离线生成后直接使用，缺少运行时上下文、技能级经验、系统化验证和可靠的长任务上下文管理。 3. 方法主线: Agent 在 ReAct 循环中先查技能目录，必要时用 skill_create 按需生成技能包，再通过测试准入、失败修补和 Skill Bank 管理完成复用闭环。 4. 技能形态: 每个技能是文件系统里的目录，核心是 SKILL.md，可带 scripts、resources、tests 等子目录，便于执行、验证和跨 Agent 迁移。 5. 技能级记忆: 每个技能旁边有 .memory.md，用来记录失败模式、输入格式注意事项和性能限制，让同一技能下次调用时带着历史经验。 6. 上下文管理: 系统把 Plan、Action、Observation 历史维护成 DAG，只压缩活跃上下文的中间部分，同时保留原始历史以便回放和恢复。 7. 实验结果: 在 51 个 SkillsBench 任务上，MUSE 加人类技能后准确率为 68.40%；在成功生成技能的 35 个任务上，自生成技能准确率达到 87.94%。 8. 迁移与效率: 把 MUSE 生成的技能直接给 Hermes 后，Hermes 从 47.89% 提升到 58.40%；在 35 个生成成功任务上，MUSE 复用生成技能比用人类技能少用约 20% token、少耗约 37% 时间。 9. 系列定位: 相比 SkillX 更关注技能知识库构建、SkillOpt 更关注外部技能文档优化、SkillClaw 更关注共享技能治理，MUSE 的重点是把创建、记忆、管理、评估和精炼统一进一个 Agent 框架。 10. 局限风险: 自生成技能依赖先跑出成功轨迹，51 个任务中只有 35 个生成成功；单条轨迹可能固化偶然条件，hvac-control 就从 80% 回落到 20%，说明测试和多轨迹提炼仍很关键。
