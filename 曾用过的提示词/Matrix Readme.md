# 项目介绍

这个是一个大型探索型项目，目前正处于起步阶段。核心目标是研究AI自主决策、自主学习、自主驱动的方法。

# 思考路径

由于AI自主是一个非常大的概念，必须缩小范围，才可以技术落地。于是抛出如下几个问题：

- 自主决策什么，怎么决策
- 自主学习什么，怎么学习
- 自主驱动什么，怎么驱动

也就是两类问题what and how

## 场景带入

假设我在经营一个电商网站，我有CDP平台，可以使用数据分析、洞察手段发现问题，使用预测模型进行推荐，可以触达客户进行沉默用户激活等，比如利用红包、礼券进行促活，我的目标就是实现ROI最大化。传统的做法是根据经验手工分析，设计营销方案，活动结束后进行分析和总结。

### what 型问题

- 自主决策什么: 分析哪些数据，生成什么报表，采用什么推荐算法，使用什么特征，采用什么触达渠道，什么触达内容。
- 自主学习什么: 
  - 人类使用工具的习惯和过程
  - 专家的思考方法和思考路径
  - 最终学会使用CDP做运营，甚至完善CDP的功能
- 自主驱动什么:
  - 驱动CDP软件的运行和使用

### how 型问题

基本思路是导师带学生的方式，假设导师是个营销和技术专家，他有多年经验生成的一套方法论，知道什么时候该做什么，每次他做动作的时候，会告诉学生，他做了什么，为什么这么做，学生可以在一个模拟环境进行实验和学习,学生可以根据实验结果提出问题，导师可以做主观回答。不断的循环上面这个过程，试试学生是否能学会导师的做事方法，然后自主决策、驱动。

# 路线图

1. 构建一个demo的电商网站
2. 构建一个模拟用户行为的服务，可以模拟用户行为去使用demo电商网站
3. 构建一个基本的CDP平台，有一些基本的功能，可以做分析、活动、触达那些模拟用户
4. 构建agent平台，可以创建一些角色，比如导师、学生的agent，让这些agent可以互相通信

# 工程结构

- website: 网站目录，放置demo电商网站、CDP平台、Agent平台
  - ecommerce: demo 电商网站的前后端工程代码等
- products: 产品文档，放置产品设计文档
  - ecommerce: demo 电商网站的产品文档
- prompts: 提示词，用于生成各种项目的提示词
  - ecommerce: demo 电商网站提示词
