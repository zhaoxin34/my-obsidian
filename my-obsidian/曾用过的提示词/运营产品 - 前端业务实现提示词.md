## 一、产品目标（你要理解的“为什么”）

这是一个 **运营策划对话式编辑系统（MVP）** 的前端。

页面的核心体验是：

> **左边是 Markdown 编辑区，右边是对话区**  
> 对话会不断修改 **同一份 Markdown 文档**

## 四、Markdown 编辑区（核心之一）

### 技术要求

- 使用 `@uiw/react-md-editor`
    
- 支持：
    
    - Markdown 编辑
        
    - 实时预览
        

### 功能要求（MVP）

- 页面加载时：
    
    - 初始化一个示例 Markdown（写死在前端即可）
        
- 提供一个「保存」按钮（占位即可）
    
- 当前阶段：
    
    - 不要求真实调用后端
        
    - 只需在前端 state 中更新 Markdown
        

---

## 五、对话区（核心之一）

### UI 要求

- 一个消息列表区
    
- 一个输入框
    
- 一个「发送」按钮
    

### 行为要求（前端模拟即可）

- 用户发送消息后：
    
    - 将消息加入消息列表
        
    - **模拟 AI 回复**
        
        - 用 `setTimeout`
            
        - 返回一段固定文本（如“已根据你的要求更新策划文档”）
            
    - 同时：
        
        - **模拟修改 Markdown 内容**
            
        - 比如在文档末尾追加一行
            

⚠️ 这一点非常重要：

> **对话 → 修改 Markdown 的链路必须在 UI 上跑通**

## 六、版本区（最小存在感即可）

### VersionPanel

- UI 形式：
    
    - 一个按钮：`版本`
        
    - 点击后：
        
        - 弹出一个 `Modal`
            
        - 显示一组假版本号（v1, v2, v3）
            
- 点击某个版本：
    
    - 模拟“回退”
        
    - 将 Markdown 内容替换为对应假内容
