我打算做一个初级版的运营产品，它的大致功能如下：
- 通过对话产生一个markdown文档，文档就是运营策划文档
- 每次对话都修改这一个markdown文档，而不是新建一个
- markdown文档也可以手动修改
- markdown每次修改都可以像git一样产生一个版本，支持回退和打版本号
UI要求：
* 页面分上中下三部分，最上面是菜单，中间区域分成左右两部分，分别是markdown编辑区和对话区，底部是一些声明区，类似绝大多数网站的风格
---
推荐一下技术框架，需要快速做出来，所以不追求大而美的框架

# ChatGPT给的前端提示词

```markdown
你是一名 **资深 React 工程师**。  
请帮我 **生成一个可运行的 React + Vite 前端工程（MVP）**，用于一个「运营策划对话式编辑系统」。

## 一、核心页面结构（必须严格遵守）

页面分为三部分：

`顶部：菜单栏 中间：左右布局   - 左：Markdown 编辑区  - 右：对话区 底部：声明 / 状态区`

---

## 二、功能要求

### 1️⃣ Markdown 编辑区

- 使用 `@uiw/react-md-editor`
- Markdown 内容来自后端 `/document`
- 用户手动修改后：
    - 点击「保存」
    - 调用 `PUT /document`

---

### 2️⃣ 对话区

- 用户输入消息
- 调用后端 `POST /chat`
- 返回的新 Markdown：
    - 自动刷新左侧编辑区
- 对话历史保留在前端即可

---

### 3️⃣ 版本管理（基础）

- 提供一个按钮：
    - 获取 `/versions`
    - 展示版本列表
    - 点击某个版本：
        - 调用 `/versions/{id}/rollback`
        - 刷新 Markdown

---

## 三、技术选型（严格）

- React
- Vite
- TypeScript
- Ant Design
- @uiw/react-md-editor
- axios

---

## 四、工程输出要求

`frontend/ ├── src/ │   ├── main.tsx │   ├── App.tsx │   ├── api.ts │   ├── components/ │   │   ├── MarkdownEditor.tsx │   │   ├── ChatPanel.tsx │   │   └── VersionPanel.tsx │   └── types.ts ├── index.html ├── package.json └── README.md`

---

## 五、工程原则

- **MVP 优先**
- 不做复杂状态管理（不用 Redux）
- 组件职责清晰
- 每个组件写清楚用途注释
```
