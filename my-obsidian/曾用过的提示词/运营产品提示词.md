我打算做一个初级版的运营产品，它的大致功能如下：
- 通过对话产生一个markdown文档，文档就是运营策划文档
- 每次对话都修改这一个markdown文档，而不是新建一个
- markdown文档也可以手动修改
- markdown每次修改都可以像git一样产生一个版本，支持回退和打版本号
UI要求：
* 页面分上中下三部分，最上面是菜单，中间区域分成左右两部分，分别是markdown编辑区和对话区，底部是一些声明区，类似绝大多数网站的风格
---
推荐一下技术框架，需要快速做出来，所以不追求大而美的框架

# ChatGPT给的后端提示词

```markdown
你是一名 **资深 Python / FastAPI 架构师**。  
请帮我 **从 0 生成一个可运行的 FastAPI 后端工程（MVP）**，用于一个「运营策划对话式编辑系统」。

---

## 一、核心业务目标

系统内 **只有一份当前生效的 Markdown 运营策划文档**，  
该文档可以通过 **LLM 对话修改** 或 **人工编辑**，  
并且 **每次修改都会生成一个新版本（类似 Git）**。

---

## 二、必须实现的功能

### 1️⃣ 文档管理

- 系统内只有一个 document
- 存储字段：
    - `id`
    - `current_content`（Markdown）
    - `current_version`
---

### 2️⃣ 版本管理（重点）

- 每次文档修改：
    - 保存一条 document_version
    - 包含：
        - `version_number`（v1, v2, v3...）
        - `content_md`
        - `diff`（使用 Python difflib 生成）
        - `created_at`
            
- 支持：
    - 获取历史版本列表
    - 回退到指定版本（并生成新版本）

---

### 3️⃣ 对话修改文档

- 提供一个 API：
    - 输入：`user_message`
    - 处理逻辑：
        1. 读取当前 Markdown
        2. 构造 LLM Prompt
        3. 调用 LLM（模型用占位符实现）
        4. 获取 **修改后的完整 Markdown**
        5. 保存为新版本
        6. 返回最新 Markdown

---

## 三、LLM Prompt（必须给出）

请在代码中实现如下 Prompt 模板（可直接替换模型调用）：

`你是一个专业的运营策划专家。 下面是当前的运营策划文档（Markdown）：  <<<CURRENT_MD>>>  用户的新要求是： <<<USER_INPUT>>>  请在尽量保持原有结构的前提下， 直接输出【修改后的完整 Markdown 文档】。 不要输出任何解释或多余文字。`

---

## 四、技术要求（严格遵守）

- Python 3.10+
- FastAPI
- SQLAlchemy 或 SQLModel
- SQLite（MVP）
- 不做登录、不做权限
- 所有接口清晰、REST 风格
    
---

## 五、接口设计（请实现）

- `GET /document`
- `PUT /document`（手动保存 Markdown）
- `POST /chat`（对话修改文档）
- `GET /versions`
- `POST /versions/{version_id}/rollback`

---

## 六、工程输出要求

请生成完整工程代码，包括：

`backend/ ├── main.py ├── database.py ├── models.py ├── schemas.py ├── services/ │   ├── document_service.py │   ├── version_service.py │   └── llm_service.py ├── requirements.txt └── README.md`

- 所有关键逻辑请加注释
- README 写清楚如何启动

---

## 七、工程原则

- **能跑优先**
- **结构清晰优先**
- 不追求抽象，不引入多余依赖
```

```markdown
你是一名 **资深 React 工程师**。

## 一、核心页面结构（必须严格遵守）

页面分为三部分：

`顶部：菜单栏 中间：左右布局   - 左：Markdown 编辑区  - 右：对话区 底部：声明 / 状态区`

    - 调用 `PUT /document`

---

### 2️⃣ 对话区

- 用户输入消息
- 调用后端 `POST /chat`
- 返回的新 Markdown：
    - 自动刷新左侧编辑区
- 对话历史保留在前端即可

---

### 3️⃣ 版本管理（基础）

- 提供一个按钮：
    - 获取 `/versions`
    - 展示版本列表
    - 点击某个版本：
        - 调用 `/versions/{id}/rollback`
        - 刷新 Markdown

---

## 三、技术选型（严格）

- React
- Vite
- TypeScript
- Ant Design
- @uiw/react-md-editor
- axios

---

## 四、工程输出要求

请生成完整工程代码，包括：

`frontend/ ├── src/ │   ├── main.tsx │   ├── App.tsx │   ├── api.ts │   ├── components/ │   │   ├── MarkdownEditor.tsx │   │   ├── ChatPanel.tsx │   │   └── VersionPanel.tsx │   └── types.ts ├── index.html ├── package.json └── README.md`

---

## 五、工程原则

- **MVP 优先**

- 组件职责清晰
```
