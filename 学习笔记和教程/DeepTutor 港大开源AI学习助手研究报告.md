# DeepTutor 港大开源 AI 学习助手研究报告

> 日期：2026-04-14
> 研究主题：香港大学数据智能实验室（HKUDS）开源的 DeepTutor 项目

## 执行摘要

DeepTutor 是由香港大学数据智能实验室（HKUDS）于 2026 年初正式开源的 AI 学习助手项目，旨在构建一个完整的"学、练、测、研"学习闭环[1]。与通用聊天机器人不同，DeepTutor 是一款聚焦教育场景的个性化学习系统，采用多智能体架构与双循环推理机制，集成了文档问答、交互式可视化、智能出题与深度研究等核心功能[2]。该项目在 GitHub 上已获得超过 9,600 颗星标[3]，采用 AGPL-3.0 开源协议，支持本地 NAS 部署，所有数据均在本地处理以保障隐私安全[4]。

DeepTutor 的核心价值在于将分散的学习工具整合为统一平台，尤其适合学生、研究人员、开发者等需深度掌握专业领域知识的人群[4]。实测数据显示，使用该系统可将知识点掌握时间从 2 小时缩短至 15 分钟，知识留存率从 20% 提升至 75%[4]。

## 一、项目概述

### 1.1 背景与定位

DeepTutor 诞生于 AI 教育工具从"问答工具"向"原生智能体导师"演进的关键阶段。回顾 AI 教育工具的发展历程，主要经历了三个阶段[5]：

- **第一阶段：问答机器人时代**。以 ChatGPT、Claude 等通用大模型为代表，可以回答任何问题，但缺乏记忆——每次对话都是全新开始，无法跟踪学习进度。
- **第二阶段：RAG 增强时代**。各种"知识库问答"系统涌现，可以基于上传的 PDF 教材进行检索，但仍是被动工具——你问它答，没有教学策略。
- **第三阶段：Agent-Native 时代**。2026 年 4 月，HKUDS 开源的 DeepTutor v1.0.0 标志着教育 AI 正式进入这一阶段。它不再是一个"带知识库的问答工具"，而是一个拥有持久记忆、主动教学策略、多实例人格的"智能体导师生态系统"[5]。

## 二、核心功能

DeepTutor 的核心功能围绕"学、练、测、研"四大模块展开，逐一击穿自学过程中的关键障碍[2]。

### 2.1 深度知识问答

DeepTutor 的文档问答功能超越了传统 RAG 工具，实现了真正的"理解与内化"[2]。用户可以上传教材、研究论文、技术手册乃至个人笔记（支持 Markdown/PDF/Word 格式），系统会通过向量模型对知识库进行内容索引，构建个人专属的 AI 驱动知识库[6]。

该系统的智能求解系统采用"分析循环 + 求解循环"的双循环架构[7]：

- **分析循环**：深入理解问题，提取关键信息
- **求解循环**：制定计划 → 调用工具（RAG 检索、网络搜索、代码执行）→ 验证答案 → 格式化输出

整个过程以实时流式输出，学习者能看到 AI 的思考过程，如同导师在白板上逐步讲解[7]。每个答案均附带精确引用，标明来自哪篇资料的哪一部分，方便追溯原始材料，这对于学术研究尤为有用[7]。

### 2.2 交互式可视化讲解

这是 DeepTutor 最具创新性的功能[2]。当用户询问复杂的数学概念或算法逻辑时，系统不会仅输出干巴巴的文字，而是直接生成一个可交互的 HTML 页面[8]。用户可以在页面上点击、拖拽，直观地观察数据和逻辑如何变化——这种将知识转化为"动态 App"的体验，比反复阅读书本更为有效[8]。

DeepTator 可将排序算法、二叉树、神经网络等抽象概念转化为直观的可视化图表，通过详细的分步分解和引人入胜的交互式演示，极大降低复杂概念的理解门槛[2]。系统还配备上下文感知对话功能，能够适应不同用户的学习进度，提供个性化的交互界面和基于会话的知识跟踪[2]。

### 2.3 智能出题与模拟训练

内置的智能出题系统能根据对话上下文和用户的知识掌握程度，个性化生成练习题[2]。该功能包含两种模式：

- **定制模式**：基于背景知识自动生成习题，支持批量生成与单轮验证[9]。
- **模拟模式**：用户上传真题试卷（PDF 格式），系统会分析其出题风格、难度分布和知识点覆盖率，生成风格相似的全新模拟卷[2]。这解决了"真题刷完无题可做"的困境，为用户提供源源不断的"仿真题"练习资源[2]。

### 2.4 深度研究模式

针对有深度探索需求的用户，DeepTutor 提供了 Deep Research（深度研究）模式[2]。该功能通过系统性分析进行深入的专题探索，能够生成结构化的学习材料并发现知识空白，助力学术研究和深度学习[2]。研究流程遵循三阶段架构：规划 → 研究 → 报告，确保研究深度和质量[9]。

DeepTutor 还可被视作 Google NotebookLM 的开源本地化替代方案[8]。它能对用户上传的海量文档进行长链路的深度分析，并生成结构化报告甚至播客讲解[8]。最关键的是，可以完全本地化部署，避免将私密笔记、未发表的论文或公司内部资料上传至云端的风险[8]。

## 三、技术架构

DeepTutor 采用四层架构设计，各层分工明确、协同运作[4]：

### 3.1 架构总览

```
┌─────────────────────────────────────────┐
│         用户界面层（Next.js 前端）         │
├─────────────────────────────────────────┤
│    智能 Agent 模块（调研、规划、解题、校验）   │
├─────────────────────────────────────────┤
│  工具集成层（TTS、网络搜索、Python 代码沙箱）  │
├─────────────────────────────────────────┤
│ 知识与记忆层（向量数据库、知识图谱、会话记忆）  │
└─────────────────────────────────────────┘
```

### 3.2 各层详解

**用户界面层**：基于 Next.js 构建，提供直观交互界面，支持双向查询-响应流程、结构化响应生成、分步问题求解和定制评估生成等功能[6]。

**智能 Agent 模块**：包含多个专业分工的智能体[4]：

- 分析代理（Investigate）：深入理解问题
- 规划代理（Plan）：制定求解计划
- 解决代理（Solve）：执行计划、调用工具
- 检查代理（Check）：验证答案准确性

**工具集成层**：支持多种工具的调用[4][6]：

- RAG 混合检索
- 实时网络搜索
- 学术论文数据库
- Python 代码执行（沙箱环境）
- PDF 文档解析
- TTS 语音合成（可选）

**知识与记忆基础层**[4][6]：

- **向量数据库**：基于嵌入（如 BGE-M3）的语义搜索，用于智能内容检索[4]
- **知识图谱**：实体-关系映射，用于语义连接和知识发现[6]
- **记忆系统**：会话状态管理和引用跟踪，确保上下文连续性[6]

### 3.3 双循环推理机制

DeepTutor 的核心推理机制为双循环架构（Dual-Loop）[8]，在回答问题前：

1. **分析回路**：先审题、查资料，如同真人教师花费大量时间备课
2. **求解回路**：分步骤执行，如同教师给学生讲课

这种机制确保了 AI 不像普通大模型那样容易"一本正经地胡说八道"，而是像真人导师一样先思考并充分准备，再给出解答[8]。

### 3.4 模型配置

值得注意的是，DeepTutor 本身不包含大语言模型，需用户自行配置[8]：

- **LLM 选项**：DeepSeek、Qwen、本地 Ollama 模型等[8]
- **向量模型**：BGE-M3 等[8]
- **TTS 和 Web 搜索**：可选组件[6]

如果用户拥有足够的本地硬件资源（如通过 Ollama 运行 Qwen），可实现完全离线的免费使用体验[6]。对于硬件条件有限的用户，也可选择调用云端 API（如 DeepSeek），对本地设备要求较低[8]。

## 四、部署与使用

### 4.1 部署方式

DeepTutor 提供多种部署方案[4][8]：

| 部署方式 | 特点 |
|---------|------|
| Docker Compose | 一键部署，推荐新手使用 |
| conda 手动安装 | 灵活定制 |
| 手动安装 | 高度自定义 |

项目适配 Windows、Linux 和 macOS 系统[4]。

### 4.2 硬件要求

- **本地大模型（Ollama）**：需具备足够 RAM 或 GPU 资源[8]
- **云端 API（DeepSeek）**：对本地设备要求较低[8]

### 4.3 NAS 部署

DeepTutor 支持在本地 NAS 设备上部署，如极空间 Z425 等[4]。这使得用户可以构建完全离线的"AI 老师"，所有数据本地处理，保障隐私安全[4]。

### 4.4 API 与费用

该项目需要使用四种 API[6]：

| API 类型 | 说明 | 费用 |
|---------|------|------|
| LLMs | 大语言模型 | 必需（可选本地 Ollama） |
| 向量模型 | 语义检索 | 必需（可选硅基流动免费 BGE） |
| TTS | 语音合成 | 可选 |
| Web 搜索 | 实时信息 | 可选 |

据实测，使用 DeepSeek API 跑完整个项目的体验测试共花费约 2 毛钱[6]。向量模型可选用硅基流动的免费 BGE，整体花费不高[6]。

## 六、项目特色总结

### 6.1 核心优势

1. **引导式教学**：系统不直接给出答案，而是通过多智能体协作分步拆解问题，确保逻辑严密并减少幻觉[8]
2. **完全本地化**：支持 NAS 部署，数据不上云端，隐私安全有保障[4]
3. **学习闭环**：整合学、练、测、研四大环节，避免工具碎片化[2]
4. **持久化导师（TutorBot）**：每个 TutorBot 拥有独立的"Soul 模板"，可定义人格、语气和教学理念，支持主动提醒和进度跟踪[5]
5. **多实例并行**：可同时运行"数学导师"和"写作教练"等多个实例，各自拥有独立记忆和工作区[5]

### 6.2 与竞品对比

| 特性 | DeepTutor | Google NotebookLM | 通用 ChatBot |
|------|-----------|-------------------|-------------|
| 本地部署 | 支持 | 不支持 | 支持 |
| 专属知识库 | 支持 | 支持 | 不支持 |
| 智能出题 | 支持 | 不支持 | 不支持 |
| 交互式可视化 | 支持 | 部分支持 | 不支持 |
| 引导式教学 | 支持 | 不支持 | 不支持 |
| 学习闭环 | 完整 | 较弱 | 无 |
