在 Bash 脚本中，处理标准输入是常见的需求。通过管道（`|`），我们可以将命令的输出传递给脚本。本文介绍两种常见的方法：`read` 命令和 `$(cat)`。

---

## 方法一：使用 `read` 命令读取输入

`read` 命令是 Bash 中常用的用于从标准输入读取数据的命令。通过管道将数据传递给脚本后，`read` 会**逐行读取**输入内容并存储到指定的变量中。

### 示例

**`test.sh` 文件内容：**

```bash
#!/bin/bash

# 使用 read 命令读取标准输入（通过管道传入的数据）
read input
echo "Received input: $input"
```

**执行命令：**

```bash
echo hello | ./test.sh
```

**输出：**

```
Received input: hello
```

### 原理解释

1. `echo hello` 会输出 `hello` 到标准输出
2. 管道 `|` 将标准输出传递给 `test.sh` 脚本
3. 脚本使用 `read` 命令读取输入并将其存储在 `input` 变量中

---

## 方法二：使用 `$(cat)` 读取输入

另一种方法是使用 `$(cat)`。它会读取**整个标准输入**并将其作为一个字符串存储。这种方法将输入内容一次性获取，适合需要将所有输入内容作为一个整体处理的场景。

### 示例

**`test.sh` 文件内容：**

```bash
#!/bin/bash

# 使用 $(cat) 来读取输入内容
input=$(cat)
echo "Received input: $input"
```

**执行命令：**

```bash
echo hello | ./test.sh
```

**输出：**

```
Received input: hello
```

### 原理解释

- `$(cat)` 会将整个输入（包括多行内容）一次性读取并存储在 `input` 变量中

---

## 两种方法对比

| 特性 | `read` | `$(cat)` |
|------|--------|----------|
| **读取方式** | 逐行读取 | 一次性读取全部 |
| **适用场景** | 按行处理数据 | 将输入作为整体处理 |
| **多行输入** | 可配合循环逐行处理 | 一次性获取所有内容 |
| **内存占用** | 较低（逐行处理） | 较高（需一次性加载） |

---

## 使用建议

- **需要逐行处理** → 使用 `read` 命令
- **需要将输入作为整体** → 使用 `$(cat)`
